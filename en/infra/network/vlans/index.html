    Homelab VLAN Design & Segmentation

    Objectif

    Cette documentation décrit l'architecture de segmentation réseau par VLANs dans le cadre d'un homelab avancé. L'approche suit les principes Zero Trust, la micro-segmentation, et une organisation par rôles fonctionnels.

    VLANs définis

    VLAN ID Nom VLAN Description
    10 Infra-Network Administration des équipements réseau : switchs, firewall, routeurs, IPMI. Utilisé exclusivement pour la gestion hors bande.
    20 Security Composants de sécurité : SIEM, IDS, log collectors, agents audit. Collecte de journaux uniquement ; aucun accès sortant.
    30 IoT-Devices Capteurs, ESP, modules connectés passifs. Aucun trafic latéral autorisé. Communique uniquement avec IoT-Services.
    31 Cameras Caméras IP, NVR. Flux vidéo uniquement. Communication permise vers stockage (NAS) ou supervision.
    33 IoT-Services Contrôleurs domotiques (Home Assistant, Mosquitto, Zigbee2MQTT). Services exposés via Ingress sécurisé.
    34 Makerspace Prototypage personnel : imprimantes 3D, OctoPrint, CNC. Accès restreint aux postes de travail de confiance.
    40 User-Trusted Postes approuvés (perso ou pro) : laptops, desktops, VMs. Accès filtré aux services internes.
    60 User-Guest Accès Internet-only pour invités. Aucune visibilité ni communication LAN.
    70 Printing Imprimantes réseau partagées. Accès via mDNS relay ou CUPS selon filtrage.
    80 NAS NAS, stockage partagé (NFS, SMB, S3). Source de sauvegarde et persistance K8s.
    90 Infra-Servers Nœuds Talos Kubernetes, API Talos, API K8s. Accessible uniquement depuis Infra-Network et User-Trusted selon règles strictes.

    Principes de conception

    • Micro-segmentation : chaque rôle réseau ou appareil a un VLAN dédié.
    • Zero Trust : aucune communication inter-VLAN autorisée sans contrôle explicite.
    • L3 Firewall Routing : tous les flux sont inspectés, filtrés et journalisés.
    • GitOps Ready : nommage exploitable dans Terraform, Ansible, NetBox, etc.

    Routage & Accès

    Flux réseau typiques

    • IoT-Devices → IoT-Services : ports MQTT, HTTP API (limités, unidirectionnels)
    • Cameras → NAS : ports NFS, SMB
    • User-Trusted → IoT-Services, Infra-Servers, Makerspace : selon ACL
    • Security → tous : collecte de logs via syslog, fluentbit, agent, NetFlow
    • User-Guest → Internet uniquement (DNS + HTTP/S sortant)

    Structuration Unifi

    Créer des profils VLAN dans Unifi Network en vous basant sur :

    • Nom : identique à la colonne "Nom VLAN"
    • VLAN ID : correspondance 1:1
    • Description : utiliser celle du tableau ci-dessus

    Regrouper dans des catégories logiques :

    • infra-*
    • iot-*
    • user-*
    • printing, security, nas

    Exemple d’assignation de port

    Port switch VLAN assigné Mode
    Port 1 Infra-Network Access
    Port 2-3 User-Trusted Access
    Port 4 Cameras Access
    Port 5 NAS Trunk
    Port 6 Infra-Servers Trunk (Talos)

    Points de vigilance

    • Ne jamais autoriser le trafic sortant de IoT-Devices, Cameras, ou User-Guest vers User-Trusted, Infra-*, ou Security.
    • Toujours loguer les accès à Infra-Servers et IoT-Services.
    • Appliquer des NetworkPolicy (Kubernetes) et firewall L3 (Unifi) pour chaque flux autorisé.

    Notes finales

    Ce modèle peut être adapté à des contextes plus larges (lab multi-site, SOC, cloud-hybride). Il est recommandé de versionner cette configuration avec les manifests K8s et les règles firewall associées.
